HTML

  • Hypertext Markup Language
  • (HTML) is one of the cornerstonelanguages used to create the framework of a webpage.
  • Made up of different elements which work together to give us the framework and content on the page.
  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Elements can aslo interact within themselfs in a child/parent relationship when nesteled (CHILD ELEMENT) (PARENT ELEMENT)
  • ELEMENT TYPES: text elements, image elements, header elements, list elements.
  • ELEMENTS THAT CAN BE SEEN ON THE PAGE: meta-related elements, content division elements, semantic Elements
  • !DOCTYPE is not case sensitive because it is a Declaration. It tells the browser what document type to expect when it's opened
  • HTML elements are case sensitive.
  • HTML attributes are always defined in the begining of the element
  • Metadata is data the webpage uses to effectivly serve our webpage.
  • Opening tags (<) Closing tags(>)

CSS

  • A padding indicates how much space we want around the content inside an element.

Git

  • git status: checks what branch we are currently on
  • git checkout -b branch-name: creates a new branch and switches to it
  • merge branches in this order:
  • 1. git add -A
  • 2. git commit -m "add your update here" (the -m is used to associate a message with the commit)
  • 3. git pull origin main (we use this to sync the local and remote branch)
  • *ALWAYS PULL BEFORE YOU PUSHING ANY CODE!*
  • 4. git push orgin feature/add-html (add-html is just and example of the branh)
  • *this command pushes all of the changes*
  • 5. Navigate to Github and the needed Repo
  • 6. click the pull request tab and add the info
  • 7. finalize the steps on git hub and close the needed issues
  • 8. Finally on the command line and update the local repo
